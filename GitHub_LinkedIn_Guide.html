Part 01 · GitHub

Create & Upload Your Repository

Follow these steps exactly, A to Z, to get a professional GitHub repo live in under 15 minutes.

01
Create a New GitHub Account (or Log In)
Go to github.com → click Sign Up if you don't have an account. Use a professional email. Your GitHub username will appear on your profile — choose something like yourname-data or just your real name.
💡 Use the same name across GitHub, LinkedIn, and your portfolio for maximum discoverability.
02
Create a New Repository
Click the + icon → New repository. Fill in the fields exactly as shown below:
Repository Settings
Repository name:   vrinda-store-data-analysis
Description:       📊 Annual sales analysis of Vrinda Store (2022) — Excel dashboard, 7 charts, business insights & recommendations
Visibility:        ✅ Public  (so recruiters & clients can see it)
Initialize with:   ✅ Add a README file  (tick this)
Add .gitignore:    None (skip)
License:           None (skip for now)

Click Create repository.

03
Replace the README with the Professional One
In your new repo, click README.md → click the ✏️ pencil edit icon. Select all existing text and delete it. Paste in the full content from the README.md file provided in this package. Then scroll down → click Commit changes.
ℹ️ The README.md from your documentation package has full badges, tables, emoji structure, and professional formatting already built in.
04
Upload All Project Files
In your repo, click Add file → Upload files. Drag and drop all files in this order:
📁 Root level: ├── Vrinda_Store_Data_Analysis.xlsx ← drag the Excel file here 📁 Create folder "docs": ├── Vrinda_Store_Documentation_Package.html ├── Presentation_Guide.md └── Stakeholder_Comms_Guide.md 📁 Create folder "data": └── data_dictionary.md
⚠️ To create a folder while uploading: type the folder name + "/" before the filename in the path field. Example: docs/Presentation_Guide.md

After uploading each batch, add a commit message like Add project documentation and click Commit changes.

05
Add Repository Topics (Tags)
On your repo homepage, click the ⚙️ gear icon next to About. Add these topics — they help people discover your project via GitHub search:
data-analysis excel dashboard pivot-tables business-intelligence ecommerce-analytics sales-analysis data-visualization india portfolio-project

Also add your website URL if you have one. Click Save changes.

06
Pin the Repository to Your Profile
Go to your GitHub profile → click Customize your pins → select vrinda-store-data-analysis. This makes it the first thing anyone sees when they visit your profile.
💡 A pinned repo on GitHub is equivalent to a featured project on a resume. Always pin your 3–6 best projects.
07
Update Your Profile README (Optional but Powerful)
Create a special repo named exactly the same as your GitHub username. The README in that repo becomes your GitHub profile page. Add a brief intro, your skills, and link to this project.

08
Copy Your Repo URL
Your repo will be live at:

📋 Post Strategy — What Makes This Work
🪝
Hook in line 1 — "the numbers tell a fascinating story" creates curiosity without revealing everything. LinkedIn's algorithm hides everything after line 3 — the hook must make people click "see more". Never start with "I'm excited to share..." — everyone does that. Start with a statement that creates a question in the reader's mind.
🔢
Specific numbers everywhere — ₹2.12 Crore, 31,047 orders, 10,740 orders, ₹75.2 Lakhs. Vague posts get ignored. Specific numbers get saved and reshared. LinkedIn data: posts with specific numbers get 3x more comments than vague ones.
📜
List format with emoji numbers — 1️⃣ through 5️⃣ makes the post scannable. Most people read the numbers first, then decide if they want the full context. Scannable = shareable. Dense paragraphs = ignored.
🎯
Business framing, not technical framing — "₹12–14 Lakhs in recoverable revenue" not "return rate was 3.4%". Recruiters and clients think in business outcomes. This framing also attracts business owners who might hire you.
🔗
GitHub link near the end — not at the top. LinkedIn's algorithm suppresses posts with external links at the top. Put the link after all the value, not before. Pro tip: some creators put the link in the first comment instead of the post itself — the algorithm ranks that even better.
#️⃣
10 targeted hashtags — mix of high-volume (#DataAnalysis, #Excel) and niche (#EcommerceAnalytics, #PortfolioProject). Don't use 30 hashtags — it looks spammy. LinkedIn recommends 3–5 hashtags. 8–10 works better in practice for discovery.
💬
End with an engagement question — "Drop a 💬 if you're working on something similar" invites comments. Comments = algorithm boost = more impressions. Posts with questions in the last line get ~40% more comments on average.
